# Security Policy

This repository is public and educational. Please do not submit private vulnerabilities, secrets, or sensitive information in public issues or pull requests.

## Reporting Sensitive Issues

If you find a security issue, contact:

**tom@tombudd.com**

Please include:

- A clear description of the issue
- Steps to reproduce, if safe
- The affected file or behavior
- Whether any secret, token, credential, or private data is involved

## Do Not Publish

Do not publish:

- Exploitable private-system details
- Secrets or credentials
- Private logs or receipts
- Personal data
- Unredacted attack chains against non-public systems

## Scope

This repository focuses on public benchmark methodology and toy examples. It should not contain private production systems. If a private artifact appears here by mistake, report it immediately.
